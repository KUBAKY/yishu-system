# 意数系统 - 云端服务器 Docker 部署方案

本文档提供了在 Linux 云端服务器（如阿里云、腾讯云、AWS 等）上通过 Docker 一键部署“意数系统”的全流程指南。

---

## 1. 前置要求

在开始部署前，请确保您的服务器已安装以下软件：
- **Docker**: (建议版本 24.0+)
- **Docker Compose**: (建议版本 2.0+)

---

## 2. 文件准备

确保项目根目录下存在以下核心文件：
1. `Dockerfile`: 定义了多阶段构建流程（已生成）。
2. `docker-compose.yml`: 定义了应用服务与数据库挂载逻辑（已生成）。
3. `.env.production`: 生产环境配置文件（需手动创建，参考下文）。

---

## 3. 环境配置

在服务器上的项目根目录下创建 `.env.production` 文件，并填入以下关键变量：

```bash
# 数据库连接（Docker 内部路径）
DATABASE_URL="file:./dev.db"

# AI 推演密钥（必须）
OPENROUTER_API_KEY="您的密钥"

# ZPay 支付商户配置（必须）
ZPAY_PID="您的商户ID"
ZPAY_KEY="您的商户密钥"

# 外部访问域名（用于支付回调）
NEXT_PUBLIC_BASE_URL="https://your-domain.com"

# 生产环境标识
NODE_ENV="production"
```

---

## 4. 部署命令

在项目根目录下执行以下命令：

### A. 构建镜像并启动服务
```bash
docker compose up -d --build
```
该命令会自动完成依赖安装、Prisma 生成、Next.js 构建以及数据库表结构初始化。

### B. 查看运行状态
```bash
docker compose ps
```

### C. 查看实时日志
```bash
docker compose logs -f yishu-app
```

---

## 5. 运维指南

### 数据库持久化
目前使用 SQLite，数据库文件存储在服务器的 `./prisma/dev.db`。Docker Compose 已将其挂载到容器内部，因此**更新应用容器不会丢失数据**。

### 停止并移除容器
```bash
docker compose down
```

### 数据库迁移（手动）
如果您修改了 `schema.prisma` 并同步到服务器，可以运行：
```bash
docker compose exec yishu-app npx prisma migrate deploy
```

---

## 6. 安全建议 (关键)
1. **反向代理**: 建议使用 Nginx 或 Caddy 对 3000 端口进行反向代理，并配置 SSL 证书。
2. **防火墙**: 服务器安全组仅需开放 80, 443 (若使用 Nginx) 或 3000 端口。
3. **备份**: 定期物理备份服务器上的 `./prisma/dev.db` 文件。
