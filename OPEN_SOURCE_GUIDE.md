### 推荐方案 C：GPL v3 (仅作为替代方案)
- **特点**: 它是强传染性的开源协议，要求任何基于此代码的商业产品也必须开源。这在很大程度上会阻止闭源商业公司的滥用。

## 2. 彻底清理敏感信息 (Security First)

在正式 `git push` 到 GitHub 公网仓库前，必须检查：

- [ ] **.env 文件**: 确认 `.env` 及其变体已在 `.gitignore` 中。确认没有在代码里硬编码任何 API KEY。
- [ ] **数据库文件**: 将 `prisma/dev.db` 加入 `.gitignore`。
- [ ] **支付信息**: 确认 `src/lib/zpay.ts` 中的 PID 和 KEY 已替换为变量读取。
- [ ] **Git 历史**: 如果之前提交过包含密钥的 Commit，简单的删除是不够的，建议使用 `bfg-repo-cleaner` 清理历史，或者干脆创建一个全新的 Git 仓库。
