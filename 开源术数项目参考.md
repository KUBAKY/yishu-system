# GitHub 开源术数项目集成参考

本文档汇总了在 GitHub 上发现的高质量中国术数（Chinese Divination）开源项目，涵盖八字、紫微斗数、周易六爻、奇门遁甲等领域。这些项目可作为后续开发智能术数推演系统的集成参考。

## 1. 综合历法与基础库 (Calendar & Utilities)
这类库是术数推演的基础，提供阴阳历转换、节气、干支计算等核心功能。

### [6tail/lunar-javascript](https://github.com/6tail/lunar-javascript)
*   **语言**: JavaScript / TypeScript (同时也支持 Java, C#, Python, Go, PHP 等多语言版本)
*   **简介**: 非常流行且功能强大的无依赖历法库。
*   **核心功能**: 
    *   公历、农历、佛历、道历转换
    *   八字排盘（年、月、日、时柱）
    *   五行、十神、纳音
    *   建除十二值星、二十八宿
    *   每日宜忌、吉神凶煞方位
*   **集成建议**: 适合作为前端或 Node.js 后端的通用历法计算引擎，数据准确且无第三方依赖。

### [wolfhong/LunarCalendar](https://github.com/wolfhong/LunarCalendar)
*   **语言**: Java / Python
*   **简介**: 农历/公历转换器。
*   **核心功能**: 包含中国传统节日和二十四节气计算。

## 2. 八字命理 (BaZi / Four Pillars of Destiny)

### [AngusHsu/lunar-mcp-server](https://github.com/AngusHsu/lunar-mcp-server)
*   **语言**: Python
*   **许可证**: MIT | ✅ 可商用
*   **简介**: 基于 **MCP (Model Context Protocol)** 的服务，专为 AI Agent 设计。
*   **核心功能**: 
    *   八字排盘与分析
    *   吉日查询、节日信息
    *   提供 MCP Tool 接口，可直接被 Claude Desktop 或其他 LLM 调用。
*   **集成建议**: **强烈推荐**。可以直接集成到当前的智能系统中，利用其现成的 MCP 接口实现“对话式排盘”和“AI 命理分析”。

### [tommitoan/bazica](https://github.com/tommitoan/bazica)
*   **语言**: Go
*   **简介**: 简单的八字计算器。
*   **核心功能**: 将公历日期转换为八字盘（四柱）。
*   **集成建议**: 如果系统后端采用 Go 语言开发，可参考此库的算法实现。

### [alvamind/bazi-calculator-by-alvamind](https://github.com/alvamind/bazi-calculator-by-alvamind)
*   **语言**: TypeScript
*   **简介**: 现代化的 Node.js 八字计算库。
*   **核心功能**: 提供完整的八字分析，包括五行强弱计算、神煞分析等。
*   **集成建议**: 适合需要深入八字逻辑（如强弱判定）的 TypeScript 项目。

## 3. 紫微斗数 (Zi Wei Dou Shu / Purple Star Astrology)

### [SylarLong/iztro](https://github.com/SylarLong/iztro)
*   **语言**: JavaScript / TypeScript
*   **许可证**: MIT | ✅ 可商用
*   **简介**: 轻量级且功能极其完善的紫微斗数排盘引擎。
*   **核心功能**: 
    *   获取紫微斗数 12 宫完整数据
    *   支持运限（大限、流年、流月、流日、流时）排盘
    *   飞星判断、四化分析
    *   支持多语言（中、英、日、韩等）
    *   支持插件扩展不同流派规则
*   **集成建议**: **首选推荐**。目前开源界最成熟的 JS 紫微斗数库，适合用于构建 Web 版排盘系统或小程序。
*   **❗ 注意**：需确认是否支持 PRD 要求的“三合/飞星/中天”多流派。插件机制可能支持扩展，但需 PoC 验证。

### [EdwinXiang/dart_iztro](https://github.com/EdwinXiang/dart_iztro)
*   **语言**: Dart / Flutter
*   **简介**: `iztro` 的 Dart 移植版。
*   **集成建议**: 如果计划开发跨平台移动端 App (iOS/Android)，可直接使用此库。

## 6. 西方占卜 (Western Divination)
GitHub 上同样存在大量高质量的西方占卜开源项目，主要集中在占星术（Astrology）和塔罗牌（Tarot）。

### 西方占星 (Western Astrology)

#### [Swiss Ephemeris (瑞士星历表)](https://github.com/aloistr/swisseph)
*   **简介**: 业界公认的占星计算“黄金标准”，提供高精度的行星位置数据。
*   **生态**: 官方为 C 语言库，但 GitHub 上有几乎所有主流语言的封装（Wrappers）：
    *   **Python**: `pyswisseph` (底层封装), `flatlib` (更易用的高层库)。
    *   **C#**: `SharpAstrology` (推荐)。
    *   **JavaScript**: `swisseph` (npm)。

#### [CReizner/SharpAstrology](https://github.com/CReizner/SharpAstrology)
*   **语言**: C# / .NET
*   **简介**: 功能强大的现代占星库。
*   **核心功能**: 
    *   基于瑞士星历表的高精度计算。
    *   支持**西方占星**、**吠陀占星 (Vedic)** 和 **人类图 (Human Design)**。
    *   提供 Blazor 组件，可快速开发 Web 排盘界面。
*   **集成建议**: 如果项目技术栈包含 .NET，这是最佳选择。

#### [theriftlab/immanuel-python](https://github.com/theriftlab/immanuel-python)
*   **语言**: Python
*   **简介**: 专为生成星盘数据设计的库。
*   **核心功能**: 计算本命盘（Natal）、推运盘（Progressions）、合盘（Synastry），包含相位、宫位等详细数据。

### 塔罗牌 (Tarot)

#### [ekelen/tarot-api](https://github.com/ekelen/tarot-api)
*   **语言**: JavaScript / JSON
*   **简介**: 提供莱德-韦特（Rider-Waite）塔罗牌数据的 REST API。
*   **核心功能**: 
    *   包含完整的 78 张牌数据（正位/逆位含义、描述）。
    *   支持随机抽牌模拟。
    *   提供结构化的 JSON 数据源，可直接下载使用。
*   **集成建议**: 适合作为塔罗牌应用的后台数据源，或直接使用其 JSON 数据进行本地开发。

#### [krates98/tarotcardapi](https://github.com/krates98/tarotcardapi)
*   **语言**: JavaScript
*   **简介**: 免费的塔罗牌 API，包含高质量的牌面图片。
*   **集成建议**: 如果需要展示牌面图片，此项目提供了很好的资源。

## 总结与开发建议

1.  **基础数据层**: 推荐统一使用 **`6tail/lunar-javascript`** (或其 Python/Java 版本) 作为底层历法引擎，确保各模块的农历数据一致。
2.  **前端/排盘展示**: 
    *   **紫微斗数**: 直接集成 **`iztro`**。
    *   **八字**: 可基于 `lunar-javascript` 或 `bazi-calculator-by-alvamind` 开发。
3.  **AI 智能化集成**: 
    *   重点关注 **`lunar-mcp-server`** 和 **`MCP-Liuyao`**。
    *   参考其 MCP 实现方式，将排盘工具封装为 AI 可调用的 Tool，实现“用户自然语言提问 -> AI 调用排盘工具 -> AI 解析盘面 -> 生成回答”的完整链路。

## 附录B：依赖冲突与兼容性分析

### B.1 同层库冲突风险

| 冲突场景 | 涉及库 | 风险描述 | 建议 |
|----------|--------|---------|------|
| **八字计算重复** | `lunar-javascript` + `bazi-calculator` | 两者皆可排八字，结果可能微差（特别是偏居部分） | 以 `lunar-javascript` 为主，`bazi-calculator` 仅作交叉验算 |
| **历法底层差异** | `lunar-javascript` vs `immanuel` | 中式农历 vs 西式星历，时区处理方式不同 | 统一时区在 API 层转换，不在库层处理 |
| **MCP 协议版本** | `lunar-mcp-server` vs `MCP-Liuyao` | 两个 MCP 服务可能依赖不同版本的 MCP SDK | 参考其接口设计，自建统一 MCP 服务 |
| **Node.js 版本** | `iztro` + `lunar-javascript` + `swisseph` | 若 Node.js 主版本不一致可能编译失败 | 统一 Node.js ≥ 18 LTS，锁定 `package-lock.json` |

### B.2 版本锁定策略

```json
// 建议的版本锁定方式
{
  "dependencies": {
    "lunar-javascript": "~1.x.x",    // 主版本锁定，允许 patch 更新
    "iztro": "~2.x.x",               // 同上
    "swisseph": "~2.x.x"             // C binding，严格锁定
  }
}
```

## 附录C：自研 vs 集成决策框架

| 能力需求 | 推荐策略 | 理由 |
|----------|----------|------|
| **历法/干支/节气** | ✅ 先集成 `lunar-javascript`，后评估自研 | 已经稳定且无依赖，自研ROI低 |
| **八字排盘** | ✅ 集成 `lunar-javascript` | 已含在历法库中，无需额外库 |
| **紫微斗数** | ✅ 集成 `iztro` | 开源界最成熟，自研工作量巨大 |
| **六爷纳甲** | ⚠️ **自研为主**，参考 `ichingshifa` 算法 | 无成熟 JS 库，且是核心能力，需掌控质量 |
| **梅花易数** | ⚠️ **自研** | 无现成库，算法相对简单 |
| **奇门遏甲** | ⚠️ **自研**，参考 `QiMenDunJia` 用例集 | C/C++ 无法直接集成，但算法可转译 |
| **大六壬** | ⚠️ **自研**，参考 `kinliuren` 规则结构 | 同上 |
| **西方星盘** | ✅ 集成 `immanuel-python` (MIT) | 避开 Swiss Eph AGPL，通过 Python 服务调用 |
| **塔罗牌** | ✅ 集成 `tarot-api` JSON 数据 | 纯数据项目，无运行时依赖 |
| **MCP 工具化** | ⚠️ **自研**，参考两个 MCP 服务的接口设计 | 统一内部 `tools.contract.json`，不绑定外部实现 |

> **决策原则**：核心业务逻辑（排盘、推演、仲裁）自研，通用基础设施（历法、星盘、数据源）集成。自研部分必须有开源库作为交叉验算基准。
